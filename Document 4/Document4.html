FEATURE SELECTION USING CORRELATION ANALYSIS:

Correlation is a bivariate analysis that determines how strong are the two variables associated with each other and the direction of their relationship. The correlation matrix shows the value of correlation coefficients between a variable with every other variable. The value of correlation co-efficient lies between –1 and 1. Any value between 0 & 1 indicates a positive relationship, whereas 0 to –1 indicates negative relationship between the two variables. Value=0 indicates there is no association between those 2 variables. We have generated a correlation matrix for our dataset using corr() function which uses Pearsons method.

Let us consider the below 3 attributes from the correlation matrix and try to visualize their relationship using scatterplot.

In a classification problem, the target variable (or output), y, can take only discrete values for given set of features (or inputs), X. In our case the target variable (Churn) is categorical and can have only 2 possible types: “0” or “1”. Thus, we use binomial logistic regression for our model. Support Vector Machines (SVM):

Support Vector Machines are basically supervised learning algorithms that is widely used for both regression and classification problems. SVM works by identifying an optimal hyper-plane that divides the two classes of labels.

SVM is used for creating both linear and non-linear boundaries that would categorize the classes. But coming to our real-time Telecom dataset being non-linear, the inputs variables are first mapped into a high dimensional space, known as Feature space, using Kernel Functions. Once the independent variables are transformed, the SVM searches for a hyper-plane that would have a maximum margin width. Margin is formed by the gap between the 2 class boundaries (positive and negative), which are defined by the most essential training tuples (Support Vectors) present on the class boundaries.

Linear SVM Classifier:

Hyperplane: It separates the data based on the linear boundary (hyperplane). The line equation of the linear boundary is:      Where x= real vector, w= normal vector to hyper-plane, b=bias.

We want to find the maximum-margin hyperplane that divides the group of input points(x) for y is 1 and from the group of points for y =-1. The maximum distance between the hyperplane and the nearest point from either group is maximised. Classification is best when the margin is wide enough. Margin is the distance between the hyper-plane and the class boundaries or nearest training tuples (support vectors) to the plane. The algorithm uses optimization procedure that focusses on maximizing the margin width.

Non-Linear SVM Classifier : When the data is non-linear, the kernel functions are used to transform inputs to high dimensional space and form a non-linear boundary. There are various types of kernel functions as given below:

When the data is highly non-linear and the plane is not able to classify the labels accurately then the algorithm has an option to allow misclassifications by using a penalty/tuning parameter (C – parameter). When the value of C is large then it maximizes the margin allowing more misclassifications, less value of C tends to decrease the width of the margin but ensuring correct classifications. Hence there is a trade-off between fitting the data and maximizing the margin.

Decision Tree:

A tree can be “learned” by splitting the source set into subsets based on an attribute value test. This process is repeated on each derived subset in a recursive manner called recursive partitioning. The recursion is continued till the subset at a node all has the same value of the target variable, or when splitting no longer adds value to the predictions. The construction of decision tree classifier does not require any domain knowledge or parameter setting.

Random Forest:

Random forest is an ensembled supervised learning algorithm which is used for both classification and regression. Random Forest classifier creates a set of decision trees from randomly selected subset of training data and gets the output/predictions from each decision tree and then selects the best prediction using the majority votes for a given class from every tree, in case of classification. Random forests are used to overcome the disadvantages of a single decision tree, i.e., Over-fitting, since it takes the average of all the tree outputs which cancels out the bias. More the numbers of trees used, more robust is the forest. It also offers a good feature selection indicator. It uses Gini Importance and Mean Decrease in Impurity (MDI) to calculate the importance of each feature. It shows by what percent the accuracy will be decreased by removing that feature. Higher decrease in accuracy indicates the feature is most significance.

PERFORMANCE EVALUATION FOR ALL MODELS:

We have split the data using the train_test_split from the model_selection. The ratio of the train and test data is 80:20(which is the ideal one) for all the models. As it splits the data randomly whenever we run the model’s multiple times, we may get the different outputs.

Confusion Matrix: A confusion matrix is used to describe the performance of a classification model or classifier typically of a supervised learning one. This is implemented on a test data where for which true values are known. It efficiently helps in measuring performance, Recall, Precision, Specificity, Accuracy and most importantly AUC-ROC Curve. It uses four different combinations of predicted and actual values, true positive, true negative, false positive and false negative.

Recall:

Out of all the positive classes, how much we predicted correctly. It should be high as possible.

Precision: 

Out of all the classes, how much we predicted correctly. It should be high as possible.

F-measure

 

In case where precision and recall values are high and low respectively, F-score helps in comparing them. Also, to Recall and Precision at the same time. It uses Harmonic Mean in place of Arithmetic Mean.

AUROC curve: AUC ROC is one of the most important evaluation metrics for any classification model’s performance. ROC (Receiver operating Characteristics) curve plots FPR versus TPR for a classifying procedure.

                                                                              

True Negative, true positive, false negative and false positive can be plotted in this manner for any given classification problem. This is how we will apply for models in our project too.

                                                     

Trade-off between sensitivity and specificity depends on the threshold we set. The curve this trade-off shows is the ROC curve. The AUC is the Area Under the Curve, for which, higher the value better the model performs. In other words, if the value is good, it means model succeeds in distinguishing positive and negative values or two required values depending on the model. Lower the value, random are the predictions. If the Area under the curve is 1, then our classifier can be proved perfect, if AUC is 0.5 it is worst possible.
